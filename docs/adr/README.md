# Architecture Decision Records

Decisions about the repo itself: structure, tooling, versioning policy, cross-language alignment rules — anything that shapes how primitives are built rather than what a specific primitive does.

Pattern-level design lives in [`../patterns/`](../patterns/), not here.

## Format

One file per decision: `NNNN-short-slug.md`, numbered sequentially. Recommended sections:

- **Status** — proposed / accepted / superseded by NNNN.
- **Context** — what forced the decision.
- **Decision** — what we chose.
- **Consequences** — what this enables and what it costs.
